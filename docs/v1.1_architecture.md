# LSMKV v1.1 Architecture

## Goal

Allow clients to connect to any node in the cluster.

The contacted node should automatically route requests
to the node responsible for the key.

---

## Planned Features

- Gateway Routing
- Request Forwarding
- Client Load Balancing
- Failure-aware Routing
- True Sharding
- Rebalancing

---

## Current Architecture (v1.0)

Client
   |
Node chosen by client
   |
Processes request

---

## Target Architecture (v1.1)

Client
   |
Any Node
   |
Routing Layer
   |
Correct Owner
   |
Storage Engine
